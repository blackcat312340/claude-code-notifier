Deliver the ERROR flood detection vertical slice: per-session error tracking with a deque-based sliding window, threshold detection (3 errors / 2 minutes), and aggregated flood notifications through the notification pipeline with a dedicated 10-minute cooldown.

**As a** developer using Claude Code/Codex,
**I want to** be notified when a session experiences an ERROR flood (3+ errors in 2 minutes),
**so that** I can diagnose hook configuration problems without monitoring individual log entries.

Purpose: Satisfies ANOM-02 (error-like abnormal state notification) and ANOM-03 (flood cooldown suppression).
Output: Working ERROR flood detection end-to-end — per-session deque tracks ERROR timestamps, threshold triggers synthetic ERROR notification with session-level 10-minute cooldown.
</objective>

| Boundary | Description |
|----------|-------------|
| ERROR events -> `_flood_tracker` deque | Per-session deque in detector.py grows with each ERROR event; must be bounded |
| Synthetic flood event -> notification pipeline | Synthetic ErrorFlood event enters the same notify_queue as hook events; must not bypass cooldown |
| should_notify conditional path | ERROR category normally blocks notification; ErrorFlood synthetic marker must be the sole exception |
