| Boundary | Description |
|----------|-------------|
| Polling timer thread -> session_registry (shared dict) | Timer callback reads registry while TCP server thread writes to it |
| Synthetic event -> notification pipeline | New event source (timer) feeds events into the same notify_queue as hook events |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Denial of Service | `_check_inactive_sessions` in detector.py | mitigate | Use `list(dict.values())` snapshot to prevent `RuntimeError: dictionary changed size during iteration` when TCP server modifies session_registry concurrently (Pitfall 3) |
| T-04-02 | Denial of Service | `datetime.fromisoformat` in detector.py | mitigate | Wrap timestamp parsing in try/except (ValueError, TypeError), skip session on parse failure, log warning with session_id (per Pitfall 3) |
| T-04-03 | Denial of Service | `_flood_tracker` growth | accept | Not introduced in this plan — handled in Plan 02 |
| T-04-04 | Tampering | Timer callback accessing freed tray resources | accept | Non-daemon Timer exits after callback returns; callback checks `tray` reference which remains valid for process lifetime |
| T-04-SC | Tampering | Package installs | mitigate | No new packages in this phase — all stdlib and existing deps only |
</threat_model>
