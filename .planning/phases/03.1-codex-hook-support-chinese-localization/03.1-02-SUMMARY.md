# Phase 03.1 Plan 02: Codex Hook Installation + Unified Config CLI Summary

**One-liner:** Codex user-level hook installation with non-destructive merge, plus unified `notifier-config install` CLI reporting both provider statuses in Chinese.

## What Was Built

### Task 1: Codex Hook Entry Builders
- Added `CODEX_HOOKS_FILE` and `CODEX_OWNERSHIP_FILE` path constants under `~/.codex`
- Added `_build_codex_hook_entries()` producing `SessionStart`, `PermissionRequest`, and `Stop` entries with provider `codex`
- Added `CODEX_HOOK_ENTRIES` and `CODEX_OWNERSHIP_MARKER` constants
- 12 tests covering entry structure, provider argument, file paths, and structural properties

### Task 2: Non-Destructive Codex Hook Installation
- Implemented `install_codex_hooks()` with the same merge pattern as Claude Code `install_hooks()`
- Strips notifier-owned entries using `_is_notifier_entry()` before merging fresh entries
- Preserves unrelated Codex settings and hook entries (D-16)
- Writes Codex ownership marker to `~/.codex/.notifier-ownership.json`
- Idempotent: running twice produces no duplicate entries
- Handles corrupt JSON gracefully (same approach as Claude path)
- 8 tests covering creation, preservation, ownership, idempotency, replacement, and error handling

### Task 3: Unified Install CLI
- Updated `notifier-config install` to call both `install_hooks()` and `install_codex_hooks()`
- Added `--codex-hooks` and `--codex-ownership` CLI options for test/advanced use
- Chinese-first provider-specific status output:
  - `Claude Code hooks: 已安装 (/path/to/settings.json)`
  - `Codex hooks: 已安装 (/path/to/hooks.json)`
- Created `test_cli_config.py` with 8 tests covering dual install, Chinese output, settings preservation, ownership files, and idempotency

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed stale test data not matching _is_notifier_entry pattern**
- **Found during:** Task 2
- **Issue:** `test_replaces_notifier_owned_codex_entries` used `"command": "old-notifier codex"` which does not contain `notifier.cli.hook`, so `_is_notifier_entry()` correctly returned False and entries were not stripped.
- **Fix:** Changed stale entries to `"/old/venv/bin/python -m notifier.cli.hook SessionStart codex"` so they match the notifier detection pattern.
- **Files modified:** src/notifier/tests/test_settings.py
- **Commit:** 0b306b1

**2. [Rule 1 - Bug] Fixed CliRunner invocation — removed subcommand name from args**
- **Found during:** Task 3
- **Issue:** All 8 CLI tests failed with "Got unexpected extra argument (install)". With a single `@app.command()`, CliRunner must not include the command name as the first argument — the command is invoked directly.
- **Fix:** Removed `"install"` from all `runner.invoke(app, [...])` argument lists.
- **Files modified:** src/notifier/tests/test_cli_config.py
- **Commit:** 819e9a1
