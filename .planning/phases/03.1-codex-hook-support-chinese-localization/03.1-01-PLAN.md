Provider-Aware Event Backbone

Extend the existing event/session/IPC model so Claude Code and Codex are first-class providers sharing one notifier pipeline. Add Codex event classification for `PermissionRequest`, `Stop`, and `SessionStart`, preserve legacy Claude Code behavior, and update session registry keys to include provider.

Purpose: establish the foundation required by all later Codex installation, UI display, and Phase 4 inactivity detection work.
Output: provider-aware event model with passing tests and no user-visible behavior regression.
</objective>

|-----------|----------|-----------|-------------|-----------------|
