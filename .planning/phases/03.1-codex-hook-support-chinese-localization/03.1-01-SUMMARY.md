# Phase 03.1 Plan 01: Provider-Aware Event Backbone Summary

**One-liner:** Added Provider enum and provider-aware classification/session/registry across the entire event pipeline, with Codex-specific event mapping and backward-compatible Claude Code defaults.

## What Changed

### Task 1: Provider enum and model fields
- Added `Provider(str, Enum)` with `CLAUDE_CODE = "claude_code"` and `CODEX = "codex"`
- Added `provider: Provider` field to `SessionInfo` and `NotifierEvent`, defaulting to `Provider.CLAUDE_CODE`
- Updated `classify_hook_event` to accept and normalize a `provider` parameter
- Updated `extract_session` to accept and propagate `provider`
- Added 10 new tests covering Provider values, defaults, string normalization, and provider propagation

### Task 2: Provider-specific classification
- Split `_classify` into a provider dispatcher calling `_classify_claude_code` or `_classify_codex`
- `_classify_claude_code`: unchanged Claude Code behavior (Notification, Stop, SessionStart)
- `_classify_codex`: maps PermissionRequest -> PERMISSION, Stop -> DONE, SessionStart -> IDLE, unknown -> ERROR
- Codex message extraction with field priority chains
- Added 11 Codex-specific classification tests; existing Claude Code tests unchanged

### Task 3: Hook CLI provider support
- Added `provider` parameter to `process_hook_event` (default `"claude_code"`)
- CLI now accepts optional second argument: `python -m notifier.cli.hook PermissionRequest codex`
- Added Codex field extraction (`hook_event_name`, `tool_name`, `final_response`) to `_safe_read_stdin`
- Added 4 Codex hook integration tests + 1 legacy backward-compat test

### Task 4: Provider-aware TCP server and session registry
- Added `provider` field to `SessionRecord` dataclass (default `"claude_code"`)
- Changed registry key format from `{session_id}||{cwd}` to `{provider}||{session_id}||{cwd}`
- `_handle_client` reconstructs provider from payload (top-level, then nested session, then default)
- `_update_session` keys by provider/session_id/cwd, avoiding Claude Code/Codex collisions
- Added 4 registry tests: key format, key separation, provider recording, collision avoidance
